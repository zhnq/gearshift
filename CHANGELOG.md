# Changelog

## 0.3.0

### 自动化与工作环境
- 场景可由程序启动/退出、前台窗口、电源、Wi-Fi、时间范围和外接显示器自动触发，所有规则有防抖保护。
- 临时场景可在其启动程序全部退出后恢复此前场景；提供 Ctrl+Alt+1..9 全局热键和桌面场景快捷方式。
- 场景支持显示器拓扑、默认播放设备和窗口布局恢复；程序库增加 Microsoft Store 与 Steam 游戏发现。

### 可观测性与性能
- 新增执行预览、失败重试和本地执行历史，每步显示耗时。
- 独立程序动作最多 3 路并发，关闭动作有 8 秒超时；程序目录改为后台缓存。

## 0.2.1

### 安全与发布
- 自动更新下载完成后强制校验 GitHub Release 资产的 SHA-256 摘要，摘要缺失或不匹配时拒绝安装。
- Release 同时发布 `SHA256SUMS.txt`，CI 支持通过仓库 Secrets 对 EXE/DLL 做可选 Authenticode 签名。
- GitHub Actions 升级到 Node.js 24/更新运行时版本，消除旧 Action 的弃用告警。

### 权限体验
- 保持普通权限启动；场景执行失败且可能需要管理员权限时，可直接“管理员重试”。
- 修复管理员重启与单实例锁竞争的问题，提权重启后自动重试刚才的场景。

## 0.2.0

### 新功能
- 启动时自动检查 GitHub 最新 Release，发现新版后由用户确认下载、覆盖并重启；设置页也可手动检查或关闭自动检查。
- 启动程序选择器整合开始菜单、桌面快捷方式和已安装程序，支持搜索、来源筛选、多选及浏览文件。
- 每个启动程序可选“正常打开”或“最小化启动”。支持托盘的程序可据自身设置进入托盘。
- 场景卡片、场景右键菜单和托盘菜单均可“再次启用”，用于重新校正用户手动改变后的程序状态。

### 修复与调整
- 完整移除不可靠的冻结/解冻功能；升级时自动清理 v0.1.2 保存的冻结条目，不影响其他场景配置。
- 设置页在宽屏和 2K 显示器上改为居中限宽布局。

## 0.1.2

### 新功能
- **场景可「冻结」程序**（声明式新增 `EnsureSuspended` 处置，实验特性）：挂起进程的全部线程（`NtSuspendProcess`），保留内存与会话但不占 CPU——游戏时冻结挂机程序，退出后原样恢复。
  - 是否已冻结由**实时探测线程状态**（`ThreadState=Wait / WaitReason=Suspended`）判定，不记 PID，无状态、跨会话可靠。
  - 切到需要该程序运行的场景会**自动解冻**；安全名单中的关键进程永不冻结。
- 场景卡片摘要显示「N 项冻结」。

> 注：冻结依赖 Windows 未公开 API `NtSuspendProcess/NtResumeProcess`，标记为实验特性。

## 0.1.1

### 修复与改进
- **添加启动程序**：文件选择器放行 `.exe / .lnk / .url / .bat / .cmd`，桌面快捷方式可直接选；`.lnk` 自动解析到目标 exe，进程名能正确参与「是否已运行」比对，不再重复启动。
- **添加关闭程序**：弹窗新增手动输入（进程名或完整路径，`.lnk` 自动解析）与「全部进程」开关，可关闭后台/托盘程序及当前未运行的程序；安全名单仍全程拦截。

## 0.1.0 (首个预览版)

首个可发布预览版：一个 Windows 桌面「场景切换」工具。

### 功能
- **声明式场景引擎**：场景 = 期望的系统状态；切换只做与当前状态的差异，切回自动恢复，幂等。
- **系统动作**：按需关闭/启动程序、开关系统代理、切换电源计划。
- **动作插件系统**：PowerShell 脚本插件，`action.json` + 脚本；支持参数、导入(.zip，含脚本信任确认)、启用/禁用。
- **托盘常驻**：右键切场景（当前场景打勾）、关窗到托盘、切换气泡通知、单实例。
- **主窗口**（Fluent，浅/深主题）：场景总览（卡片/新建/快照新建/右键编辑删除）、场景编辑（程序增删·全局动作·插件动作·图标）、动作库、设置。
- **设置**：主题、开机自启、默认场景（启动自动应用）、关键进程安全名单、以管理员身份重启。
- **安全**：关键系统进程永不被关闭。

### 已知限制
- 示例插件「专注助手」「默认音频设备」为演示占位脚本，未实现真实系统切换（`clean-ram` 为真实实现）。
- 插件动作仅 apply 相位，暂不做 read 状态比对。
- 尚无自定义应用图标；托盘为字母图标。
